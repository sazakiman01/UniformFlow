# Team Protocol

*กติกาการทำงานของทีม (ดู `team/team-config.md` สำหรับ Role mapping)*

## Handoff งานระหว่าง Role

1. บันทึก status ปัจจุบัน
2. ระบุ Role ปลายทาง + context ที่ต้องรู้
3. บันทึก handoff ลง `team/daily/YYYY-MM-DD.md`

## งานที่ต้องการหลาย Role

1. ระบุ Primary + Supporting (ดู team-config)
2. กำหนด responsibility + timeline ของแต่ละ Role
3. บันทึก `team/daily/YYYY-MM-DD.md`

## Resolve Conflict

1. ฟัง perspective ทุก Role
2. วิเคราะห์ trade-offs
3. ตัดสินใจด้วย reasoning ชัดเจน
4. บันทึก `Luna/decisions.md` + `team/daily/`

## Review งาน

1. เลือก Role reviewer (เช่น Sirius สำหรับ QA, Vega สำหรับ UX)
2. ให้ context ชัดเจน
3. รับ feedback → ปรับปรุง
4. บันทึก `team/daily/`

## สร้าง Role ใหม่

1. วิเคราะห์ด้วย `team/skill-gap-analysis.md`
2. ใช้ `team/onboarding-template.md` สร้าง Role
3. ตั้งชื่อ Space Theme
4. สร้าง `team/roles/[name].md` + `team/preferences/[name].md`
5. Update `team/team-config.md` (Quick Lookup)
6. บันทึก `team/daily/`

## Performance Review

- **รายสัปดาห์:** Review `team/performance.md` → `Luna/retrospective/weekly-YYYY-Www.md`
- **รายเดือน:** Retrospective ทั้งทีม → `Luna/retrospective/monthly-YYYY-MM.md`
