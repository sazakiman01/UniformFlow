# Team Performance Tracking

*ส่วนนี้ track ว่า Role ไหน perform ดี/ไม่ดีในงานประเภทไหน*

## รูปแบบการบันทึก

```
### [Role Name] - [YYYY-MM-DD]

**งานที่ทำ:** [ชื่องาน]
**ประเภทงาน:** [architecture/backend/frontend/database/etc]
**ผลลัพธ์:** [สำเร็จ/ไม่สำเร็จ]
**ความคิดเห็น:** [วิเคราะห์ performance]
**ปรับปรุง:** [สิ่งที่ควรปรับปรุง]
```

## Performance Log

*ยังไม่มี performance log*

## การใช้ Performance Log

- Review รายสัปดาห์/เดือน
- ใช้ปรับปรุง prompt ของ Role
- ติดตามว่า Role ไหน perform ดี/ไม่ดี
- พิจารณา restructure team หรือสร้าง Role ใหม่
