### ปรับ Contrast หน้า Business Model Canvas ให้อ่านง่ายขึ้น

**ปัญหา:**
- หน้า `/dashboard/admin/business-model` ดูกลืนกับพื้นหลัง
- อ่านแล้วสว่างแสบตาเกินไป
- Background cards ใช้สี pastel จางมาก (opacity /40) กลืนกับ `bg-slate-50/50`
- Border colors จางเกินไป (blue-100, emerald-100 ฯลฯ)
- Text colors มี contrast ต่ำกับ background ที่จาง

**วิธีแก้ (Hybrid Approach):**
1. **เพิ่ม opacity background cards** จาก `/40` → `/80` (ชัดขึ้นแต่ยังคงสี pastel)
2. **เพิ่มความเข้ม border** จาก `100` → `200` (ชัดขึ้น)
3. **เพิ่ม contrast text colors:**
   - TONES object: text จาก `700` → `800`, iconBg จาก `100` → `200`
   - BlockCard: list items `slate-700` → `slate-800`, empty state `slate-400` → `slate-500`
   - FinanceBlockCard: item names `slate-800` → `slate-900`, amounts `slate-900` → `slate-950`
4. **เพิ่ม subtle shadow:** เพิ่ม `shadow-sm` เป็น base class และ `hover:shadow-md` สำหรับทั้ง BlockCard และ FinanceBlockCard

**ผลลัพธ์:**
- Cards ชัดขึ้น ไม่กลืนกับ background
- Border เห็นชัดเจนขึ้น
- Text อ่านง่ายขึ้น ไม่ต้องจ้องมอง
- ยังคงสีสันที่สวยงาม (ไม่เปลี่ยนเป็นขาวล้วน)
- เปลี่ยนแปลงน้อยที่สุด ไม่กระทบ design มาก

## Learnings

- Contrast ที่ต่ำเกินไปทำให้ eye strain และอ่านยาก
- Hybrid approach (เพิ่ม contrast แต่ยังคงสีสัน) เป็นทางเลือกที่ดีกว่าเปลี่ยนเป็นขาวล้วน
- Shadow ช่วยแยก elements จาก background ได้ดี
- Tailwind color scale 100-200-300-400-500-600-700-800-900-950 มีความสำคัญกับ accessibility
- Confirm dialog สำคัญมากสำหรับ destructive actions (ลบข้อมูล)
- ใช้สีแดง (rose-600) สำหรับปุ่มลบเพื่อสื่อความเป็น action อันตราย
- Backend function มีอยู่แล้วแต่ไม่มี UI คือปัญหาที่พบบ่อย
- Tooltip ช่วยมือใหม่เข้าใจฟิลด์ได้ดีโดยไม่ทำให้ UI รบกวน
- ใช้ native tooltip (title attribute) เพราะ simple และไม่ต้อง install dependency เพิ่ม
- Lucide icons ไม่รองรับ title prop โดยตรง ต้อง wrap ใน span หรือ div
