## 🔍 Investigation

### Root Cause
- `CustomerPicker.tsx` บรรทัด 215: set `email` เป็น `undefined` เมื่อเป็น empty string
- Firestore ไม่รองรับค่า `undefined` ใน document fields โดยตรง
- เกิด error เมื่อ `addDoc()` พยายามบันทึก document ที่มี field `email: undefined`

### Files Analyzed
- `d:\coding\UniformFlow\src\lib\customers.ts` — createCustomer function
- `d:\coding\UniformFlow\src\components\admin\CustomerPicker.tsx` — modal สำหรับเพิ่ม/แก้ลูกค้า
- `d:\coding\UniformFlow\src\components\admin\QuotationForm.tsx` — form ที่เรียกใช้ CustomerPicker

---

## ✅ Solution

**Approach:** Sanitize data ก่อนส่งไป Firestore โดย filter undefined values ออก

**Implementation:**
- แก้ใน `src/lib/customers.ts` function `createCustomer`
- เพิ่ม `sanitizedData` ที่ filter undefined values:
  ```typescript
  const sanitizedData = Object.fromEntries(
    Object.entries(data).filter(([_, v]) => v !== undefined)
  );
  ```

**Benefits:**
- แก้ปัญหาที่ source (createCustomer function)
- ป้องกันปัญหาเดียวกันจากที่อื่นที่เรียก createCustomer
- เป็น best practice ในการ sanitize data ก่อนส่ง database
