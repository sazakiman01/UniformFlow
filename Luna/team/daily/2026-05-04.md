### Task 6: แก้ปัญหา Firestore permissions เมื่อลบ catalog

**ปัญหา:**
- Error "Missing or insufficient permissions" เมื่อลบ catalog
- Firestore rules รวม update และ delete ไว้ด้วยกัน และ validate data ด้วย `isValidCatalogData(request.resource.data)`
- Delete operation ไม่มี `request.resource.data` (เป็น null) ทำให้ validation fail

**วิธีแก้:**
- แยก rule สำหรับ delete ออกมาไม่ต้อง validate data
- เปลี่ยนจาก `allow update, delete: if isWarehouse() && isValidCatalogData(request.resource.data);`
- เป็น:
  - `allow update: if isWarehouse() && isValidCatalogData(request.resource.data);`
  - `allow delete: if isWarehouse();`

**ผลลัพธ์:**
- User ที่มี role warehouse สามารถลบ catalog ได้แล้ว
- Firestore rules deploy สำเร็จ

**ไฟล์ที่แก้ไข:**
- `firebase/firestore.rules` (บรรทัด 173-174)

**Deployment:**
